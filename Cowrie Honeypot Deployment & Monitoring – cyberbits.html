Cowrie Honeypot Deployment & Monitoring

November 16, 2025

Ahoneypotis acybersecurity decoy systemdesigned to look like a real computer, server, or servicebut its only purpose is toattract attackers,record their actions, andstudy their techniques.

In this blog, I will show you how I manage to create a honey pot in my Linux VM and convey how attackers can connect to your network and potentially harm it.

In this scenario, I will be using Cowrie (link included) a medium interaction SSH and Telnet honeypot, which can log data from attackers including IP addresses, brute-force attacks, location etc..

Feel free to download and try this one on your own!

Tools Used: Note: I already downloaded the requires packages and programs needed to run Cowrie. (Check the gift hub link)

  • Linux Ubuntu VM
  • Cowrie Honeypot program
  • MacOS (for testing the connection) or another kind of endpoint.

Starting the Honeypot Lab

Boot up the VM and head on in to the Terminal. I type the command of:

$ docker run -p 2222:2222 cowrie/cowrie:latest

This tells Docker tostart the Cowrie honeypot containerandmap port 2222 from the container to port 2222 on the host machine. This means anything that connects to your computer on port2222will actually be talking to the Cowrie honeypot running inside Docker.

This screenshot showsCowrie is fully running and activeinside Docker and is awaiting SSH connections so anyone can try to connect to this network.

Next, find the IP address of the endpoint that is running Cowrie. This will be the host needed to connect to from my Mac.

To find the IP address, type:

hostname -I

you will be presented with the endpoint’s credentials the. 192.168.x.x is the one we need to connect to.

In MacOS

We hop on over to the MacOS terminal where we can now test the honeypot with this command:

ssh admin@192.168.x.x -p 2222

  1. The Secure Shell (SSH) command used to remotely log into another computer.
  2. admin@192.168.x.x The username isadmin, and the machine youre connecting to has the IP192.168.x.x
  3. -p 2222 is the port number of this connection, therefore, my Mac is attempting to connect to this port.

Once we input the command on the Terminal, it prompts us for the password to successfully connect. Let’s input some random phrases.

  • ADMIN123
  • PASSWORD123
  • THECAKEISALIE123

On the Linux VM:

Here what was captured:

  • The Time and date
  • The IP address of the attacker (192.168.x.x)
  • the passwords attempted (ADMIN123, PASSWORD123, THECAKEISALIE123)

This is an example of a brute-force attack where many passwords are tried and fail. Attackers may possess programs that input common phrases and words at lighting speed to try to break in.

Researching and Remediation Tactics

  • We can look up the origin the IP address to known the location of where the attack is coming from. (AbuseIPBD, Talos, etc..)
  • Study the patterns of the passwords used and implore users/employees to create a more complex password with various combinations of characters, symbols, numbers etc. to avoid common phrases. and reduce the likelihood of breach.
  • Implement a locking system where it can mitigate brute-force attacks whether that be a forced password reset after 5 attempts, 2 step verification etc..

Exposing the Honeypot Publicly

Now with the honeypot built and understood, we can expose it to real-world attackers and bots where they can poke around and peruse.

NOTE: Remember to take necessary precautions when exposing your honeypot online. Lack of negligence can lead attackers a way into your network. Thankfully, there are safety measures we can implement.

Run it in a DMZ

DMZ (Demilitarized Zone)means placing it in aspecial, isolated network segmentwhere outside attackers can access it but it cannot reach or harm your internal network.

I already have:
Cowrie running inside Docker
Docker running inside an Ubuntu VM
Ubuntu VM running on your Mac
Cowrie exposed only on port 2222(which we will change to port 22 for further authenticity)

This seems sufficient enough!

Change to Port 22(SSH)

Change from port 2222 -> 22

Stop & remove the existing Cowrie container

docker stop cowrie
docker rm cowrie

This clears the old container so you can re-run it mapped to the new port.

Start Cowrie mapped to port 22

docker run -d –name cowrie -p 22:2222 cowrie/cowrie:latest

This means:

  • The world connects toport 22
  • Docker forwards traffic into the container on2222
  • Cowrie still runs internally on 2222 (its default)
  • Port 2222 on your host is nowfree to use

Alright, now the ssh is running and awaiting connection! Let it run and come back later.

Here is an example of results I’ve gotten before. Notice the IP addresses are from bots of different countries and popular passwords were also used. Highlighted some.

Popular passwords used globally are:

  • admin
  • password
  • 123456
  • qwerty

I got passwords such as oracle and support in this example. Very interesting.

Final Thoughts

This was an exciting and fun project to work on. Not only was I able to construct a honeypot on my own machine and think like one, but I actually got to test it with real-world attackers/bots and discover more about their methodology and tactics. The information I gathered can be very helpful to aspiring SOC analysts, cybersecurity students or bit tech corporations!

There are many things you can do now with a honeypot such as forwarding to a SIEM and creating alerts on credentials used. The possibilities are endless. Thank you for reading!
